HTMLの基礎

HTMLファイルの作り方

HTMLファイルとは，メモ帳のようなテキストファイルだが，次の特徴をもつ．

  • （通常は）ブラウザを利用して閲覧する．
  • メモ帳とは異なり，文字や色をつけたり，背景画像を設定したり，ボタンを配置したりと，様々なことができる．
  • JavaScriptと連携できる．
  • 実は，文法にミスがあっても自動修正で表示できてしまう（メリットでありデメリットでもある）

HTMLファイルの作り方は非常に簡単である．

  1. テキストファイルを開き，何か文章を書く．
  2. ファイルを保存し，拡張子を.htmlに変更する．
  3. ダブルクリックで開く．

これだけで，webページの作成が完了する．現段階は，ただのプレーンテキストであり無味乾燥な文章しか表示されないが， これにタグと呼ばれる記法を利用して文字や領域に様々な属性を付与することにより，色々な表現が可能となる．

HTMLは前述の通り，文法に従わなくてもブラウザが自動的に修正してしまう．したがって，本来は正しく色々と 書かないといけないのだが，このページはHTMLの手軽さ及びタグという概念を学ぶことを目的とするため， 文法的に正しく書くことは行わない．それについては，後のページで扱う．

タグ

htmlでは，「タグ」と呼ばれるブロックで文字列を囲むことにより，その文字列になんらかの属性を付与することができる． ここで，タグとは，<と>で囲まれたものを指す．例えば， 

<xxx>sample</xxx>

とすれば，「sample」という文字列に「xxx」という属性が付与されたことになる． sampleの左側にあるタグ<xxx>のことを「Start tag（開始タグ）」などといい，sampleの右側にあるタグ</xxx>のことを 「End tag（終了タグ）」などという． 一方，タグの中には文字列に属性を付与する以外が目的のものがある．例えば「ここで改行を入れる」のようなものは， 文字列への属性付与とは無関連である．そのようなタグは単に 

<xxx>

とだけ記述する．このようなタグのことを「Void element」という． タグ内の「xxx」に書くことができる文字列には色々とあり，これらをこのページで最小限学んでいく．

タグには上記のように2種類あるため，文法的に一貫性に欠け，美しくない． そこで，「すべてのタグは閉じるべきである」という信念を導入した言語が存在する（例：xhtml）． そのような言語では，Self-closingタグは，開始タグと終了タグをまとめた<xxx/>という形で書くよう決められている．
タグには詳細設定の情報を付与することができる．例えば， <xxx>というタグに，「optionA」という詳細設定と「optionB」という詳細設定の情報を付与したい場合は，<xxx option1 option2>のように記載する．

注意事項

HTMLは，テキストファイルに書いた文書をそのまま表示するわけではない．特に，最初は以下のことに注意しておくとよい：

  • 単一の改行は無視される
  • 連続した空白・改行・タブは1つの空白として処理される
  • タグが特別な意味をもつことから，< や > はエスケープする必要が生じてくる．

例：

"Say,                       Tom, 
where are


you going?"
                  "Nowhere."

表示：

Say, Tom, where are you going?" "Nowhere."

このことから，改行を表現したいときは，何らかの形でタグの力を借りる必要が生じてくる．

タグの種類1

タグには様々な種類があるが，何回かに分けて紹介していく．

記法 解説
<br> 改行を入れる．
<strong> ～ </strong> 囲まれた部分を太字にする．
<p> ～ </p> 囲まれた部分を一つの段落とみなして表示する．（自動改行処理が入る）
<a href="リンク先"> 文字列 </a> 文字列をクリックすると，指定したリンク先に飛ぶ機能を付ける．

使用例：

<p>
  HTMLでは，このように簡単に文章を表示することができます．<br>
  ただし，改行は<strong>brタグ</strong>を利用する必要があります．
</p>
<p>
  <strong>pタグ</strong>は段落ブロックを意味するので，それを使っても改行が表現できます．<br>
  改行が無視されるというのは最初は不便に思えるかもしれませんが，
  インデントの調整によってコードの構造が明確になるため，実は便利な仕様です．
</p>
<p>
  HTMLについては，<a href="https://google.com">google</a>で検索しても良いでしょう．
</p>

特別な記法

タグは特別な意味合いをもつため，不等号記号をHTML文章内で文字として入力する際は，タグと誤認されないようにエスケープする必要がある． HTMLでもいくつかエスケープ文字が用意されている．また，通常のプログラミング言語にあるように，コメントもつけることができる． エスケープ文字は大量にあるため，必要なものに絞って最低限のみ説明する．

記法 解説 目的
<!-- コメント入力部分 --> コメントとして取り扱われ，htmlの表示には反映されない． 何か備忘録を残したい時に使う．
&lt; 「<」を出力する．「<」（小なり）は英語で less than なので，その頭文字を用いる． タグとの混同を防ぐため．
&gt; 「>」を出力する．「>」（大なり）は英語で greater than なので，その頭文字を用いる． タグとの混同を防ぐため．
&amp; 「&」を出力する．「&」は英語で ampersand なので，その頭文字を用いる． エスケープ文字との混同を防ぐため．

例：

<!-- 不等式と真偽 -->

2 < 3 は真であり，5 > 7 は偽である．したがって，(2 < 3) && (5 > 7) は偽である．
エスケープ文字は，必ず使わないといけないようなものではなく，誤認される時に使わないといけないものなのだが， 使っておくことが推奨される．

タグの種類2

ここでは，このページで取り扱う残りの補助的なものを紹介する．

記法 解説 補足
<button> ～ </button> ボタンを配置する． 単独では意味がない．buttonタグにオプションを追加して，javascriptのトリガーとして使用する．下の節で扱う．
<input> テキストボックスを配置する． 単独では意味がない．inputタグにオプションを追加して，ユーザーからの入力を javascript に渡したり， javascript から値を表示したりなどの目的で使用する．下の節で扱う．
<div> ～ </div> 囲まれた領域に名前を付ける際に使用するコンテナ．領域は，何を指定してもよく，別のタグ領域が入っていてもよい． 名前を付けることで，javascriptから呼び出して操作することが可能になる．
<span> 文字列 </span> 囲まれた文字列領域に名前を付ける際に使用するコンテナ．領域は，divとは異なり，文章の一部を指定する． 名前を付けることで，javascriptから呼び出して操作することが可能になる．
class="クラス名" divやspanを含め，タグ領域に対してクラス名を付ける．クラス名は，初心者のうちは，「半角英数」および「アンダースコア」を利用するとよい． クラス名とは，グループ名のようなもので，複数の領域に同じクラス名を付与することが可能． また，一つの領域に複数のクラスを宛がうことも可能．
id="id名" divやspanを含め，タグ領域に対してid名を付ける．クラス名は，初心者のうちは，「半角英数」および「アンダースコア」を利用するとよい． id名とは，個人名のようなもので，同じid名は一つの領域のみに付与することが可能．

使用例1：

<p>
  span領域は文字列をspanタグで<span>囲む</span>ことで指定できます．
  クラス名も設定することができます．<br>
  例えば，クラス名を test1 とすれば，
  <span class="test1">このように</span>設定できます．idも同様です．
</p>
<p>
  ボタン<button class="test1">ボタン</button>や
  テキストボックスにも<input id="textBox">クラス名やid名を付けることができます．
</p>

使用例2：


<p>
  div領域は，任意領域をdivタグで囲むことで指定できます．
</p>
<div class = "test1">
  <p>
    div領域の中には，pタグブロックを含めることもできます．同様に，class名も指定できます．
  </p>
  <div id = "name1">
    <p>
      id名の指定も同様です．
      このように，div領域の中にさらにdiv領域を入れることができ，
      実際はなんでも入れることが可能です．
    </p>
  </div>
  <div class = "test2" id = "name2">
    <p>
      このように，classとidを両方指定することもできます．
    </p>
  </div>
</div>
