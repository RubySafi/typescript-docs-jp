TypeScript とは

TypeScript とは，JavaScript に型の概念を付与した言語である． 特に，紹介する関数の中には JavaScript に固有の関数（たとえば console.log）が含まれる． TypeScript での解説であっても，基本的な機能は JavaScript の標準的な関数として紹介することがある．

コメントとログ出力

Javascriptで標準で用意されている組み込み関数console.logを用いて，値をコンソールに表示することができる． 主な使い方は，console.log(値)である．一方，ソースコードには，コードに影響を与えない任意の文字列（コメント）を残すことができる． なお，JavaScriptでは，各命令（文）の末尾には「;（セミコロン）」を付けるのが基本である．

  • 「//」の後ろに書いたものはすべてコメントになる．ただし，同一行に限る．
  • 「/*」と「*/」で囲まれた範囲はコメントになる．複数行も可能である． 

          console.log(345); // 345と出力される．
          /*
          console.log(456); ここはコメントなので，何も起こらない．
          */

変数の宣言

値は変数を宣言したあと，「=」を用いて代入することができる．型に関する指定や，より厳密な宣言方法については次の節で扱うこととし，ここではもっとも基本的な代入の方法を紹介する． constおよびletが代表的な宣言方法である．constは再代入が不可な変数を宣言する際に用い， letは再代入可能な変数を宣言する際に用いる． 

const a = 3;    // a という変数に3を代入した．aの値は，以後変更しない．
console.log(a); // 3 と表示される．

let b = 4;      // b という変数に4を代入した．値は後で書き換えることができる．
console.log(b); // 4 と表示される．

b = 5;          // b の値を 5 に書き換える．const として b を宣言しているとエラーになる．
console.log(b); // 5 と表示される．
文字列は，"abc"や'abc'のようにクォートで囲むやり方が基本である．（他にもある）これを使えば， 
let c = "hoge";
のように文字列を代入できる．
変数の宣言をする前に代入しようとすると，エラーを吐く．先に宣言だけして，後で代入することは認められる．

値の比較と変数の宣言

二つの値が完全に等しいか否かは，===（等しい）および!==（等しくない）により判定できる． 一方，問うまでもない比較式は，文法上は正しいが，設定によっては静的解析ツール（ESLintなど）が警告やエラーを出すことがあるため，注意が必要である． 正しいときはtrue，等しくないときはfalseが出力される． 


console.log(345 === 345); // trueと出力される．

let a, b;                 // 文字列をまとめて宣言．後で代入する．
a = "abc";
b = "def";
console.log(a === b);     // a と b は等しくないので false が出力される．
console.log(a !== b);     // a と b は等しくないので true が出力される．
明らかに等しくないものは，設定次第でエラーを吐くことがある． 
console.log(345 === 234);   // 明らかに等しくないので，エラーとなることがある．

console.log(345 === '123'); // 左辺は数字，右辺は文字で，比較するまでもなく
console.log(345 !== '123'); // 等しくないので，エラーとなることがある．
以下，設定依存でエラーを出すものについては，細かく扱わないものとする．
